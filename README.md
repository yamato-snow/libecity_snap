# Libecity Snap

> イベントの写真共有をより簡単に、楽しく。ゲストはQRをスキャンして写真を撮影・アップロード、全員の視点が1つのアルバムに集まります。

## プロジェクト概要

### コンセプト

イベントでゲストが撮った写真はLINE、AirDrop、SNSとバラバラに散らばる。
Libecity Snapは「QRスキャン → PIN入力 → ニックネーム登録 → 撮影/アップロード」で、
全ゲストの写真を1つのアルバムに集約する。

### 開発目的

| 目的 | 内容 |
|-----|------|
| **個人的動機** | 自分のイベント（2027-01-31）で使う |
| **ビジネス目的** | なし（個人利用のみ、商用ではない） |
| **学習目的** | Next.js App Router + Turso/Prisma + S3 Presigned URL + Amplifyの実践 |

### プロジェクトの位置づけ

- [ ] 学習プロジェクト
- [x] MVP（最小実装）→ イベント当日に1回だけ本番利用

## ターゲットユーザー

### 管理者（主催者）

```
【イベントを主催する主催者】として
【ゲスト全員の写真を1箇所に集めて閲覧・ダウンロード】を利用したい
【なぜなら】イベント後にバラバラに写真をもらうのが大変だから
```

### ゲスト（50〜100人）

| 項目 | 内容 |
|-----|------|
| **年齢層** | 20〜60代 |
| **特徴** | スマホリテラシーは様々。Snow/Ulike等の加工アプリ利用者もいる |
| **ニーズ** | アプリ不要で簡単にアップロードしたい |

```
【イベントに参加するゲスト】として
【スマホで撮った写真を簡単にアップロード】を利用したい
【なぜなら】アプリのインストールや会員登録は面倒だから
```

## 主要機能

### ゲスト向け機能

| 機能 | 説明 | 状態 |
|-----|------|------|
| PIN認証 | QRスキャン後に共通4桁PIN入力。案内にQR+PINを印刷 | 実装済 |
| 参加者名検索 | PIN認証後に参加者リストから名前検索で本人を特定 | 実装済 |
| ニックネーム登録 | 参加者と紐づけてニックネーム登録。UNIQUE制約 + 1参加者1ゲスト制約。localStorage+Cookie保持 | 実装済 |
| 再ログイン | 参加者名検索→ニックネーム照合でセッション復元 | 実装済 |
| ブラウザカメラ撮影 | capture属性でカメラ起動→即アップロード | 実装済 |
| ギャラリーアップロード | Snow/Ulike等で撮った写真を選択してUP（複数枚OK）。キュー+リトライ付き | 実装済 |
| HEIC→JPEG変換 | iPhoneのHEIC画像をクライアント側で自動変換 | 実装済 |
| シーン選択 | 管理者設定のシーン（イベント/懇親会/フリータイム等）を選んでUP | 実装済 |
| 共有アルバム閲覧 | シーン別・投稿者別フィルタ + 無限スクロール | 実装済 |
| 写真モーダル表示 | アルバムの写真をタップで拡大表示 | 実装済 |

### 管理者向け機能

| 機能 | 説明 | 状態 |
|-----|------|------|
| 管理者認証 | 別PINで管理者としてログイン | 実装済 |
| 統計ダッシュボード | 写真数・ゲスト数・シーン別集計 | 実装済 |
| シーン管理 | シーンの追加・削除（写真紐づきチェック付き） | 実装済 |
| 写真管理 | 単体削除・選択モードでの一括削除 | 実装済 |
| ゲスト管理 | 単体削除・一括削除（関連写真のカスケード削除） | 実装済 |
| 参加者管理 | CSV一括登録・個別追加・削除 | 実装済 |
| 一括ダウンロード | 全写真をZIPでダウンロード | 実装済 |
| 当日のみアップロード制限 | `UPLOAD_RESTRICTION_ENABLED=true` で有効化、`NEXT_PUBLIC_EVENT_DATE` 当日のみUP可能（JST判定） | 実装済 |

### 未実装機能

| 機能 | 説明 |
|-----|------|
| ゲスト個別ダウンロード | 気に入った写真を選んでDL |
| サムネイル自動生成 | 一覧表示の高速化 |

### UX対策

| 課題 | 対策 |
|------|------|
| iOS Safari Cookie消失（ITP） | localStorage優先で保持。再ログインフローで復帰可能 |
| 会場の電波不良 | アップロードキュー + 指数バックオフリトライ（最大5回、2並列） |
| ニックネーム重複 | Turso UNIQUE制約で確実にチェック |
| guestId偽装 | Bearer tokenのguestIdをDB照合して実在確認 |
| ニックネーム忘れ | 管理画面で参加者ごとのニックネームを確認可能 |
| アルバム表示・フィルタ | カーソルベースページネーション + Prismaクエリ |
| 操作が分からないゲスト | PIN入力画面に簡単な操作ガイドを表示 |

## 技術スタック

> 詳細な選定理由は [ADR-001-技術選定.md](./docs/ADR-001-技術選定.md) を参照

### 構成

```
フロントエンド: Next.js 16.1.6 (App Router) + React 19 + TypeScript 5 + Tailwind CSS 4
バックエンド:   Next.js API Routes（フロントと一体型）
DB:            Turso (libSQL / ホスト型SQLite) + Prisma 6 ORM
ストレージ:     AWS S3（写真ファイルのみ、Presigned URL）
インフラ:       AWS Amplify（git pushデプロイ、Docker不要）
```

### 主要ライブラリ

| ライブラリ | 用途 |
|-----------|------|
| `@prisma/adapter-libsql` | Turso接続（PrismaLibSql） |
| `@aws-sdk/client-s3` | S3操作 |
| `@aws-sdk/s3-request-presigner` | Presigned URL生成 |
| `heic2any` | HEIC→JPEG変換（クライアント側） |
| `jszip` | ZIP一括ダウンロード |
| `uuid` | 写真IDの生成 |

### アーキテクチャ概要

```
ゲスト端末 (スマホブラウザ)
    │ HTTPS
    ▼
Amplify: Next.js (App Router)
    ├─ /            ゲスト用メイン画面（PIN → 参加者検索 → ニックネーム登録 → アップロード）
    ├─ /album       共有アルバム（フィルタ + 無限スクロール）
    ├─ /admin       管理画面（統計 / 写真管理 / ゲスト管理）
    └─ /api/        API Routes (14エンドポイント)
           │              │
           ▼              ▼
     Turso (libSQL)    S3 (写真のみ)
     - Invitee         photos/{uuid}.jpg
     - Guest (→Invitee)
     - Scene
     - Photo
     via Prisma
```

### 環境変数

> 詳細は [環境変数リファレンス](./docs/05_環境変数リファレンス.md) を参照

| 変数名 | 説明 |
|--------|------|
| `EVENT_PIN` | ゲスト認証用PIN（4桁） |
| `ADMIN_PIN` | 管理者認証用PIN（4桁） |
| `TURSO_DATABASE_URL` | Turso データベースURL（`xxx`含むとローカルSQLiteにフォールバック） |
| `TURSO_AUTH_TOKEN` | Turso 認証トークン |
| `AWS_ACCESS_KEY_ID` | AWS IAMアクセスキー |
| `AWS_SECRET_ACCESS_KEY` | AWS IAMシークレットキー |
| `AWS_REGION` | S3バケットのリージョン（例: `ap-northeast-1`） |
| `S3_BUCKET_NAME` | 写真保存先S3バケット名 |
| `UPLOAD_RESTRICTION_ENABLED` | `true` で当日のみアップロード許可 / `false` で常時許可 |
| `NEXT_PUBLIC_EVENT_DATE` | イベント日 `YYYY-MM-DD`（アップロード制限判定 + クライアント表示に使用） |

**アップロード制限の仕組み:**
- `UPLOAD_RESTRICTION_ENABLED=false`（デフォルト）: いつでもアップロード可能（開発・テスト用）
- `UPLOAD_RESTRICTION_ENABLED=true`: `NEXT_PUBLIC_EVENT_DATE` と一致する日（JST）のみアップロード可能
- 本番当日に Amplify の環境変数で `true` に切り替えて再デプロイする運用

## プロジェクト構成

```
src/
├── app/
│   ├── page.tsx              # メイン画面（PIN → ニックネーム → アップロード）
│   ├── album/page.tsx        # 共有アルバム
│   ├── admin/page.tsx        # 管理画面
│   └── api/
│       ├── auth/             # POST: PIN認証
│       ├── guests/           # GET/POST: ゲスト一覧・登録（参加者紐づけ必須）
│       │   └── verify/       # POST: 再ログイン検証（nickname + inviteeId照合）
│       ├── invitees/         # GET/POST: 参加者一覧・登録（管理者のみ）
│       │   ├── search/       # GET: 参加者名検索（registeredOnlyフィルタ対応）
│       │   ├── batch/        # POST: CSV一括登録（管理者のみ）
│       │   └── [id]/         # DELETE: 参加者削除（管理者のみ）
│       ├── photos/           # GET: 写真一覧（カーソルページネーション）
│       ├── upload/           # POST: Presigned URL取得 + Photo作成
│       ├── scenes/           # GET/POST: シーン一覧・追加
│       │   └── [id]/         # DELETE: シーン削除
│       └── admin/
│           ├── download/     # GET: 全写真URL一覧
│           ├── photos/[id]/  # DELETE: 写真削除
│           └── guests/       # GET: ゲスト一覧
│               └── [id]/     # DELETE: ゲスト削除（カスケード）
├── components/               # 17コンポーネント
├── hooks/
│   ├── use-session.ts        # localStorage + Cookie セッション管理
│   ├── use-upload-queue.ts   # アップロードキュー（2並列、指数バックオフ）
│   └── use-online-status.ts  # オンライン/オフライン検知
├── lib/
│   ├── auth.ts               # PIN検証、Bearer認証、Cookie fallback、guestId DB照合
│   ├── prisma.ts             # Prisma + Turso接続（dev時はローカルSQLite）
│   ├── s3.ts                 # Presigned URL生成、S3オブジェクト削除
│   └── constants.ts          # 定数（ファイルサイズ上限、イベント日等）
└── types/
    └── heic2any.d.ts         # heic2anyの型定義
scripts/
└── test-auth.ts              # 認証セキュリティE2Eテスト（30項目）
```
