# 要件定義書: Libecity Snap

## 基本情報

| 項目 | 内容 |
|------|------|
| プロジェクト名 | Libecity Snap |
| 作成日 | 2026-03-11 |
| イベント日 | 2027-01-31 |
| ビジョン | ゲストがQRをスキャンして写真を撮影・アップロードし、全員の視点が1つのアルバムに集まる |
| 利用形態 | 個人利用（自分のイベント専用、商用ではない） |

---

## 1. 課題と解決

### 1.1 現状の課題

| # | 課題 | 痛み度 |
|---|------|--------|
| 1 | ゲストが撮った写真がバラバラ（LINE、AirDrop等）で集まらない | ★★★★★ |
| 2 | 「あの写真送って」のやりとりが面倒 | ★★★★ |
| 3 | 誰が撮ったか、いつのシーンかが分からなくなる | ★★★ |

### 1.2 解決後の姿

- 全ゲストの写真が**1つのアルバム**に自動で集まる
- **投稿者名**と**シーン**が紐付いて整理された状態で閲覧できる
- ゲストも自由に写真をダウンロードして持ち帰れる

---

## 2. ターゲットユーザー

### 2.1 管理者（主催者）

```
【イベントを主催する主催者】として、
【ゲスト全員の写真を1箇所に集めて閲覧・ダウンロード】したい。
なぜなら【イベント後にバラバラに写真をもらうのが大変】だから。
```

### 2.2 ゲスト（50〜100人）

```
【イベントに参加するゲスト】として、
【スマホで撮った写真を簡単にアップロード】したい。
なぜなら【アプリのインストールや会員登録は面倒】だから。
```

**ゲストの特徴:**
- スマホリテラシーは様々（20代〜60代）
- アプリインストールはハードルが高い → **ブラウザ完結**が必須
- Snow、Ulike等の加工アプリを使う人もいる

## 3. 機能要件（MoSCoW分類）

### Must（これがないと価値ゼロ）

| ID | 機能名 | 説明 | 完了条件 |
|----|--------|------|----------|
| M1 | PIN認証 | QRスキャン後に共通4桁PINを入力してアクセス。案内にQR+PINを印刷 | PINが正しい場合のみ次の画面に遷移する。不正PINではエラー表示 |
| M2 | ニックネーム登録 | PIN認証後にニックネームを自由入力。リアルタイム重複チェックで一意性を担保。localStorage + Cookieで保持し再入力不要。セッション消失時はニックネーム再入力で既存セッション復元可能 | 未登録ニックネームが受理され、重複時は「すでに使われています」と表示される。2回目以降は入力不要。ブラウザ再起動後もlocalStorageで復帰できる |
| M3 | ブラウザカメラ撮影 | スマホブラウザからカメラを起動して撮影→即アップロード | capture属性でカメラが起動し、撮影した写真がアップロードされる |
| M4 | ギャラリーアップロード | 端末のギャラリーから写真を選択してアップロード（複数枚対応）。アップロードキュー+リトライ機能付き。失敗時は端末に保持し電波回復後に自動再送 | Android/iOS両方で、ギャラリーから1枚以上の写真を選択してアップロードできる。電波不良時もキューに保持され、回復後に自動アップロードされる |
| M5 | 投稿者名の自動紐付け | 登録済みニックネームが自動で写真に紐付く（手動入力不要） | アップロードされた写真に認証済みニックネームが紐付いている |
| M6 | シーン選択 | アップロード時に管理者が設定したシーンを選択 | シーン一覧が表示され、選択したシーンが写真に紐付く |

### Should（あると便利）

| ID | 機能名 | 説明 | 完了条件 |
|----|--------|------|----------|
| S1 | 共有アルバム閲覧 | 認証済みユーザーがシーン別・投稿者別に写真を閲覧 | フィルタ切り替えで写真が絞り込まれる |
| S2 | ゲスト個別ダウンロード | ゲストが気に入った写真を選択してダウンロード | 写真を選択してDLボタンを押すとダウンロードされる |
| S3 | 管理者一括ダウンロード | 管理者が全写真をZIPでダウンロード | /admin画面から一括DLボタンで全写真がZIPでDLされる |
| S4 | 当日のみアップロード制限 | イベント日（2027-01-31）のみアップロード可能 | イベント日以外にアップロードしようとするとメッセージが表示される |

### Could（余裕があれば）

| ID | 機能名 | 説明 |
|----|--------|------|
| C1 | 管理画面でシーン管理 | Web UIからシーンを追加・編集・削除 |
| C2 | サムネイル自動生成 | アップロード時にサムネイルを生成して一覧表示を高速化 |
| C3 | 写真の並び替え | アップロード日時順、シーン順で並び替え |

### Won't（今回はやらない）

| ID | 機能名 | 理由 |
|----|--------|------|
| W1 | 本格的なユーザー登録/ログイン | 個人利用なので不要。PIN+ニックネームで十分 |
| W2 | イベント作成機能 | 1回きりの利用なので不要 |
| W3 | リアルタイム通知 | 過剰。アルバムを見に行けば十分 |
| W4 | 写真フィルター/加工 | Snow等の外部アプリで対応 |
| W5 | SNS連携 | スコープ外 |
| W6 | コメント/いいね機能 | 過剰 |

## 5. 非機能要件

### 5.1 アップロード

| 項目 | 要件 |
|------|------|
| アップロード速度 | 1枚あたり5秒以内（Presigned URLによるS3直接アップロード） |
| 画質 | オリジナル画質を保持（サーバー側でのリサイズ・圧縮なし） |
| 対応フォーマット | JPEG, PNG, HEIF/HEIC（アップロード時にJPEG変換） |
| HEIF/HEIC対応 | クライアント側でJPEG変換後にアップロード（heic2anyライブラリ） |
| ファイルサイズ上限 | 1枚あたり10MB（異常ファイルの除外用） |
| 電波不良対策 | アップロードキュー + 自動リトライ。失敗時は端末に保持し回復後に自動再送 |

### 5.2 表示・読み込み

| 項目 | 要件 |
|------|------|
| 表示最適化 | next/image による動的リサイズ・WebP変換・キャッシュ（S3にはオリジナル1枚のみ保存） |
| アルバム一覧 | サムネイルサイズで軽量表示。オリジナル画像を直接読み込まない |
| 写真詳細 | 画面幅に合わせたサイズで表示。DL時はオリジナル画質を取得 |
| 対応端末 | スマホブラウザ（iOS Safari、Android Chrome） |
| 同時アクセス | 50〜100人が同時にアクセスしても動作する |

### 5.3 セッション

| 項目 | 要件 |
|------|------|
| セッション保持 | localStorage + Cookie併用。iOS SafariのITP対策としてlocalStorage優先 |
| セッション復元 | ニックネーム再入力で既存セッション復帰可能（TursoのGuestテーブルで照合） |
| セキュリティ | PIN認証 + ニックネーム登録の2段階。URLを知っていてもPINがなければアクセス不可 |

### 5.4 運用

| 項目 | 要件 |
|------|------|
| 写真保持期間 | イベント後3ヶ月（その後S3から削除してローカル保存） |

## 6. 技術スタック

| 項目 | 技術 | 選定理由 |
|------|------|----------|
| フレームワーク | Next.js（App Router） | UI/UX作り込みに最適。フロント+APIを1つにまとめてシンプルに |
| 言語 | TypeScript | 型安全 |
| DB | Turso（libSQL / ホスト型SQLite） | SQLite互換、HTTP接続でサーバーレス対応、コールドスタートなし、無料枠9GB |
| ORM | Prisma + @prisma/adapter-libsql | 型安全なクエリ、UNIQUE制約で重複チェック確実、マイグレーション管理 |
| ストレージ | AWS S3 | Presigned URLで写真を直接アップロード。写真ファイル(.jpg)のみ格納 |
| デプロイ | AWS Amplify | Next.js App Routerネイティブ対応。git pushで自動デプロイ。Docker不要 |
| スタイリング | Tailwind CSS | ユーザーファーストなUI/UX作り込みに最適 |

## 7. データモデル

メタデータはTurso（Prisma）で管理。写真ファイルのみS3に格納。

### Prismaスキーマ

```prisma
model Guest {
  id        Int      @id @default(autoincrement())
  nickname  String   @unique
  createdAt DateTime @default(now())
  photos    Photo[]
}

model Scene {
  id        Int     @id @default(autoincrement())
  name      String
  sortOrder Int     @default(0)
  photos    Photo[]
}

model Photo {
  id          String   @id @default(uuid())
  guest       Guest    @relation(fields: [guestId], references: [id])
  guestId     Int
  scene       Scene    @relation(fields: [sceneId], references: [id])
  sceneId     Int
  storagePath String
  uploadedAt  DateTime @default(now())
}
```

### S3バケット構成

```
libecity-snap-bucket/
└─ photos/
    └─ {uuid}.jpg    ← 写真本体のみ
```

### データの役割分担

| データ | 格納先 | 理由 |
|--------|--------|------|
| ゲスト情報 | Turso（Guestテーブル） | UNIQUE制約で重複チェック確実。同時書き込み安全 |
| シーン情報 | Turso（Sceneテーブル） | 管理画面からCRUD可能 |
| 写真メタデータ | Turso（Photoテーブル） | SQLクエリでフィルタ・ソートが容易 |
| 写真ファイル | S3 | 大容量バイナリはオブジェクトストレージが最適 |

## 8. アーキテクチャ概要

```
┌──────────────────────────────────────────────────┐
│  ゲスト端末（スマホブラウザ）                       │
│  QRスキャン → PIN認証 → ニックネーム登録             │
│  → シーン選択 → 撮影 or ギャラリー選択 → UP         │
│                                                   │
│  [localStorage + Cookie でセッション保持]            │
│  [アップロードキュー: 失敗時は保持→自動リトライ]      │
└──────────────┬───────────────────────────────────┘
               │ HTTPS
┌──────────────▼───────────────────────────────────┐
│  Amplify: Next.js（App Router）                    │
│  ├─ /            ゲスト用メイン画面                 │
│  ├─ /album       共有アルバム                      │
│  ├─ /admin       管理画面                          │
│  └─ /api/                                         │
│      ├─ POST /api/auth       PIN認証（環境変数照合）│
│      ├─ POST /api/guests     ニックネーム登録       │
│      ├─ POST /api/upload     Presigned URL発行      │
│      ├─ GET  /api/photos     写真一覧取得           │
│      └─ GET  /api/scenes     シーン一覧取得         │
└──────────────┬───────────────────────────────────┘
       ┌───────┴───────┐
       ▼               ▼
┌──────────────┐ ┌──────────────┐
│  Turso       │ │  S3           │
│  (libSQL)    │ │               │
│              │ │  photos/      │
│  - Guest     │ │    {uuid}.jpg │
│  - Scene     │ │               │
│  - Photo     │ │               │
│  via Prisma  │ │               │
└──────────────┘ └──────────────┘
```
