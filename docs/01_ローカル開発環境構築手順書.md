# 01. ローカル開発環境構築手順書

> Libecity Snap をローカルで動作させるための環境構築手順

## 前提条件

| 項目 | 要件 |
|------|------|
| **Node.js** | v20 以上（v22 推奨） |
| **npm** | v10 以上（Node.js 同梱） |
| **Git** | インストール済み |
| **OS** | Windows / macOS / Linux |

### Node.js のインストール確認

```bash
node -v   # v20.x.x 以上であること
npm -v    # v10.x.x 以上であること
```

未インストールの場合は [Node.js 公式サイト](https://nodejs.org/) から LTS 版をダウンロードしてインストールしてください。

---

## 手順

### Step 1: リポジトリのクローン

```bash
git clone https://github.com/<your-username>/libecity-snap.git
cd libecity-snap
```

### Step 2: 依存パッケージのインストール

```bash
npm install
```

主要な依存パッケージ:

| パッケージ | 用途 |
|-----------|------|
| `next@16.1.6` | フレームワーク |
| `@prisma/client` + `prisma` | ORM（DB操作） |
| `@prisma/adapter-libsql` | Turso接続アダプタ |
| `@aws-sdk/client-s3` | S3操作 |
| `@aws-sdk/s3-request-presigner` | Presigned URL生成 |
| `heic2any` | HEIC→JPEG変換 |
| `jszip` | ZIP生成（管理者一括DL） |

### Step 3: 環境変数ファイルの作成

プロジェクトルートに `.env` ファイルを作成します。

```bash
cp .env.example .env  # テンプレートがある場合
```

テンプレートがない場合は、以下の内容で `.env` を作成してください:

```env
# === 認証 ===
EVENT_PIN=1234
ADMIN_PIN=9999

# === データベース（ローカル開発ではダミー値でOK） ===
TURSO_DATABASE_URL=libsql://libecity-snap-xxx.turso.io
TURSO_AUTH_TOKEN=dummy

# === AWS S3（ローカル開発でもS3は必要） ===
AWS_ACCESS_KEY_ID=<S3手順書で取得した値>
AWS_SECRET_ACCESS_KEY=<S3手順書で取得した値>
AWS_REGION=ap-northeast-1
S3_BUCKET_NAME=<S3手順書で作成したバケット名>

# === イベント設定 ===
UPLOAD_RESTRICTION_ENABLED=false
NEXT_PUBLIC_EVENT_DATE=2027-01-31
```

> **ポイント**: `TURSO_DATABASE_URL` に `xxx` が含まれている場合、アプリは自動的にローカルSQLite（`prisma/dev.db`）にフォールバックします。ローカル開発ではTursoのセットアップは不要です。

各環境変数の詳細は [05_環境変数リファレンス.md](./05_環境変数リファレンス.md) を参照してください。

### Step 4: ローカルデータベースのセットアップ

#### 4-1. Prisma Client の生成

```bash
npx prisma generate
```

#### 4-2. ローカルSQLiteにスキーマを反映

```bash
npx prisma db push
```

これにより `prisma/dev.db` が作成され、Guest / Scene / Photo テーブルが作成されます。

#### 4-3. 初期データの投入（シード）

```bash
npm run db:seed
```

以下の3つのデフォルトシーンが作成されます:

| ID | シーン名 | sortOrder |
|----|---------|-----------|
| 1 | イベント | 1 |
| 2 | 懇親会 | 2 |
| 3 | フリータイム | 3 |

#### 4-4. データベースの確認（任意）

Prisma Studio でデータを確認できます:

```bash
npm run db:studio
```

ブラウザで `http://localhost:5555` が開き、テーブルの内容をGUIで確認・編集できます。

### Step 5: 開発サーバーの起動

```bash
npm run dev
```

起動後、以下のURLでアクセスできます:

| URL | 画面 |
|-----|------|
| `http://localhost:3000` | ゲスト用メイン画面 |
| `http://localhost:3000/album` | 共有アルバム |
| `http://localhost:3000/admin` | 管理画面 |

### Step 6: 動作確認

1. `http://localhost:3000` にアクセス
2. PIN入力画面で `1234` を入力
3. ニックネームを入力して登録
4. シーンを選択して写真をアップロード

> **注意**: 写真アップロードにはS3の設定が必要です。S3未設定の場合、アップロード時にエラーになります。S3の構築手順は [02_AWS_S3構築手順書.md](./02_AWS_S3構築手順書.md) を参照してください。

## 利用可能な npm スクリプト一覧

| コマンド | 説明 |
|---------|------|
| `npm run dev` | 開発サーバー起動（ホットリロード有効） |
| `npm run build` | プロダクションビルド（Prisma generate + Next.js build） |
| `npm run start` | プロダクションサーバー起動（build後に使用） |
| `npm run lint` | ESLint実行 |
| `npm run db:migrate` | Prismaマイグレーション作成・適用 |
| `npm run db:push` | スキーマをDBに直接反映（マイグレーションファイルなし） |
| `npm run db:seed` | シードデータ投入 |
| `npm run db:studio` | Prisma Studio起動（DB GUIツール） |

## トラブルシューティング

### `prisma generate` でエラーが出る

```
Error: @prisma/client did not initialize yet.
```

→ `node_modules` を削除して再インストールしてください:

```bash
rm -rf node_modules
npm install
npx prisma generate
```

### ポート3000が既に使われている

```bash
# 使用中のプロセスを確認（Windows）
netstat -ano | findstr :3000

# 別ポートで起動
npx next dev -p 3001
```

### HEIC画像のアップロードが失敗する

`heic2any` はブラウザ上で動作するライブラリです。Node.js環境（サーバーサイド）では動作しません。ブラウザから操作してください。
