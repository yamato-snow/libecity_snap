# 05. 環境変数リファレンス

> Libecity Snap で使用するすべての環境変数の一覧と設定ガイド

---

## 環境変数一覧

| 変数名 | 必須 | 例 | 説明 |
|--------|------|-----|------|
| `EVENT_PIN` | Yes | `1234` | ゲスト認証用PIN（共通） |
| `ADMIN_PIN` | Yes | `9999` | 管理者認証用PIN |
| `TURSO_DATABASE_URL` | Yes※ | `libsql://libecity-snap-user.turso.io` | Turso データベースURL |
| `TURSO_AUTH_TOKEN` | Yes※ | `eyJ...` | Turso 認証トークン（JWT） |
| `AWS_ACCESS_KEY_ID` | Yes | `AKIA...` | AWS IAMアクセスキーID |
| `AWS_SECRET_ACCESS_KEY` | Yes | `wJa...` | AWS IAMシークレットアクセスキー |
| `AWS_REGION` | Yes | `ap-northeast-1` | S3バケットのAWSリージョン |
| `S3_BUCKET_NAME` | Yes | `libecity-snap-photos` | S3バケット名 |
| `UPLOAD_RESTRICTION_ENABLED` | Yes | `false` | アップロード日制限の有効/無効 |
| `NEXT_PUBLIC_EVENT_DATE` | Yes | `2027-01-31` | イベント日（サーバー・クライアント共用） |

> ※ `TURSO_DATABASE_URL` に `xxx` が含まれる場合、ローカルSQLite にフォールバックするため、ローカル開発ではダミー値でも動作します。

---

## 各変数の詳細

### 認証関連

#### `EVENT_PIN`

| 項目 | 内容 |
|------|------|
| 用途 | ゲストがアプリにアクセスする際のPIN認証 |
| 使用箇所 | `src/lib/auth.ts` の `verifyPin()` |
| 推奨値 | 4桁の数字（案内に印刷する） |
| 開発時の値 | `1234` |
| 本番での注意 | 推測されにくい番号を設定。イベント前に変更推奨 |

#### `ADMIN_PIN`

| 項目 | 内容 |
|------|------|
| 用途 | 管理画面（`/admin`）へのアクセス認証 |
| 使用箇所 | `src/lib/auth.ts` の `verifyAdminPin()` |
| 推奨値 | 4桁の数字（`EVENT_PIN` とは異なる値） |
| 開発時の値 | `9999` |
| 本番での注意 | 主催者のみが知る番号を設定 |

### データベース関連

#### `TURSO_DATABASE_URL`

| 項目 | 内容 |
|------|------|
| 用途 | Turso リモートデータベースへの接続URL |
| 使用箇所 | `src/lib/prisma.ts` の `PrismaLibSql` アダプタ |
| 形式 | `libsql://<db-name>-<username>.turso.io` |
| 取得方法 | `turso db show <db-name> --url` |
| 開発時の値 | `libsql://libecity-snap-xxx.turso.io`（`xxx`を含むとローカルSQLiteにフォールバック） |

#### `TURSO_AUTH_TOKEN`

| 項目 | 内容 |
|------|------|
| 用途 | Turso データベースの認証トークン |
| 使用箇所 | `src/lib/prisma.ts` の `PrismaLibSql` アダプタ |
| 形式 | JWT（`eyJ...`） |
| 取得方法 | `turso db tokens create <db-name>` |
| 開発時の値 | `dummy`（ローカルSQLiteフォールバック時は使用されない） |
| 有効期限 | デフォルトは無期限。`--expiration 90d` で期限指定可能 |

### AWS S3 関連

#### `AWS_ACCESS_KEY_ID`

| 項目 | 内容 |
|------|------|
| 用途 | S3 へのプログラムアクセス認証 |
| 使用箇所 | `src/lib/s3.ts` の `S3Client` |
| 形式 | `AKIA` で始まる20文字の文字列 |
| 取得方法 | IAMコンソール > ユーザー > セキュリティ認証情報 > アクセスキーを作成 |

#### `AWS_SECRET_ACCESS_KEY`

| 項目 | 内容 |
|------|------|
| 用途 | S3 へのプログラムアクセス認証（秘密鍵） |
| 使用箇所 | `src/lib/s3.ts` の `S3Client` |
| 取得方法 | IAMアクセスキー作成時に1回だけ表示される |
| 注意 | 紛失した場合は新しいキーペアを再発行 |

#### `AWS_REGION`

| 項目 | 内容 |
|------|------|
| 用途 | S3バケットのリージョン指定 + 画像URLの生成 |
| 使用箇所 | `src/lib/s3.ts`, `next.config.ts` |
| 設定値 | S3バケットの実際のリージョンと完全一致させること |
| 確認方法 | `aws s3api get-bucket-location --bucket <bucket-name>` |
| よくあるエラー | リージョン不一致で301リダイレクトが発生 → 画像が表示されない |

#### `S3_BUCKET_NAME`

| 項目 | 内容 |
|------|------|
| 用途 | 写真の保存先S3バケット名 |
| 使用箇所 | `src/lib/s3.ts`, `next.config.ts` |
| 命名規則 | 小文字英数字とハイフンのみ、3〜63文字、グローバルで一意 |

### イベント設定関連

#### `UPLOAD_RESTRICTION_ENABLED`

| 項目 | 内容 |
|------|------|
| 用途 | アップロードを `NEXT_PUBLIC_EVENT_DATE` 当日のみに制限するかどうか |
| 使用箇所 | `src/app/api/upload/route.ts` |
| 設定値 | `true` = 当日のみ許可 / `false` = 常時許可 |
| 推奨 | 開発・テスト時: `false` / 本番当日: `true` |
| タイムゾーン | JST（日本標準時）で判定 |

#### `NEXT_PUBLIC_EVENT_DATE`

| 項目 | 内容 |
|------|------|
| 用途 | イベント日（アップロード制限の判定 + クライアント表示に使用） |
| 使用箇所 | `src/app/api/upload/route.ts`, `src/lib/constants.ts` |
| 形式 | `YYYY-MM-DD` |
| 注意 | `NEXT_PUBLIC_` プレフィックスにより、ビルド時にクライアントコードに埋め込まれる |

## 環境別の設定ガイド

### ローカル開発（`.env`）

```env
EVENT_PIN=1234
ADMIN_PIN=9999
TURSO_DATABASE_URL=libsql://libecity-snap-xxx.turso.io
TURSO_AUTH_TOKEN=dummy
AWS_ACCESS_KEY_ID=<実際のキー>
AWS_SECRET_ACCESS_KEY=<実際のキー>
AWS_REGION=ap-northeast-1
S3_BUCKET_NAME=libecity-snap-photos
UPLOAD_RESTRICTION_ENABLED=false
NEXT_PUBLIC_EVENT_DATE=2027-01-31
```

### 本番環境（Amplify 環境変数）

```env
EVENT_PIN=<本番用PIN>
ADMIN_PIN=<管理者用PIN>
TURSO_DATABASE_URL=libsql://libecity-snap-<user>.turso.io
TURSO_AUTH_TOKEN=<実際のトークン>
AWS_ACCESS_KEY_ID=<実際のキー>
AWS_SECRET_ACCESS_KEY=<実際のキー>
AWS_REGION=ap-northeast-1
S3_BUCKET_NAME=libecity-snap-photos
UPLOAD_RESTRICTION_ENABLED=true
NEXT_PUBLIC_EVENT_DATE=2027-01-31
```

## セキュリティ上の注意

| 項目 | 対策 |
|------|------|
| `.env` ファイル | `.gitignore` で除外済み。**絶対にGitにコミットしない** |
| シークレットの共有 | Slack やメールで平文送信しない。1Password 等のパスワードマネージャーを使用 |
| アクセスキーのローテーション | 定期的にキーを再発行し、古いキーを無効化する |
| PINの強度 | 4桁PINはブルートフォースに弱いが、用途（1日限りのイベント）から許容範囲 |
